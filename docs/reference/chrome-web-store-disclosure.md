# Chrome Web Store Data-Use Disclosure

The extension listing and privacy disclosure must state the following before the article-saving release is submitted:

> When you explicitly choose “Save article to Docket,” the extension reads the rendered page and sends a cleaned copy of the article’s full text and HTML, source and canonical URLs, title, author, description, site name, publication date, image and favicon URLs, language, word count, and extraction duration to your authenticated Docket account. Docket uses this data only to provide private saving, version history, reading, excerpt association, and search. Docket does not send page content to a third-party extraction service. Publisher-hosted HTTPS media may load when you read the saved copy, but Docket does not archive those media files.

The disclosure should classify full article bodies and saved excerpts as website content/user-provided content, and source URLs as browsing-related content collected only after an explicit save action. Authentication data is used to associate a save with the user’s private account. The listing must not claim offline media archiving.

Deletion wording must explain that deleting an article removes every stored article version while retaining previously saved excerpts as unlinked records. Account-level retention and deletion statements must remain consistent with the public privacy policy.
